# On Writing While Building

I want this section to be lightweight enough that I actually use it. That means the bar for publishing cannot be "write a polished essay." It has to support short notes, partial conclusions, and ideas that are still forming.

Writing is useful because it exposes weak reasoning early.

## Why keep a public log

1. It creates a record of how my thinking changes.
2. It turns vague opinions into claims that can be checked later.
3. It makes it easier to share what I am learning without packaging everything as a project case study.

## What I want from the format

The format should be simple:

- Open a markdown file.
- Add a title, date, excerpt, and tags.
- Write the post.

That is enough for a first version. If the writing habit becomes real, then the publishing system can become more sophisticated later.
